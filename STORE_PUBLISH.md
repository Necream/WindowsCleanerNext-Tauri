# Microsoft Store Publish Guide

This repository can be submitted to Microsoft Store as a Win32 app.

## Recommended route

- Keep the current Windows installer flow first
- Submit the packaged Win32 app through Partner Center
- If you later want a more store-native experience, migrate to MSIX

Microsoft supports Win32 app distribution through the Microsoft Store, and the app package must remain stable after submission.

## What you need before submitting

1. A Microsoft developer account
2. Access to Partner Center
3. A reserved app name
4. App metadata:
   - name
   - description
   - category
   - privacy policy URL
   - screenshots
   - icon
5. A Windows installer package:
   - `MSI`
   - or `EXE` installer
6. A submission draft:
   - [STORE_LISTING.md](./STORE_LISTING.md)
   - [PRIVACY_POLICY.md](./PRIVACY_POLICY.md)

## What this repository already has

- Windows release bundles generated by Tauri
- GitHub Release automation for tag pushes
- Versioned release assets

## Suggested submission flow

1. Sign in to Partner Center.
2. Reserve the app name.
3. Create a new app submission.
4. Fill in store listing details.
5. Upload the installer package.
6. Complete the age rating and certification steps.
7. Submit for review.

## Files you will likely reuse

- `src-tauri/target/release/bundle/msi/*.msi`

- [STORE_LISTING.md](./STORE_LISTING.md)

## Notes

- Store submission still requires your own Microsoft account.
- If you want the app to look more native in Store, the next step is converting to MSIX.
- If you want, this repository can later be extended with a dedicated MSIX workflow.
